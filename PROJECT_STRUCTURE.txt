FEATURE BREAKDOWN
=================

Audio Services:
- audio_service.py: Handles audio file I/O and manipulation
- audio_stitching.py: Stitches multiple audio segments together
- tts_service.py: Converts text to speech using XTTS v2

Text & PDF Processing:
- text_service.py: Text normalization and preprocessing
- pdf_service.py: Extracts text from PDF documents

NLP Features:
- dialogue_service.py: Detects and extracts dialogue from text
- emotion_engine.py: Analyzes emotional content
- speaker_service.py: Identifies speakers in text

API Layer:
- routes.py: Defines REST API endpoints

Configuration:
- config.py: Centralized configuration management
- .env.example: Environment variable template


DATA FLOW
=========

User PDF Upload
    ↓
pdf_service.py (Extract text and metadata)
    ↓
text_service.py (Normalize and preprocess)
    ↓
dialogue_service.py (Detect dialogue vs narration)
    ↓
speaker_service.py (Identify speakers)
    ↓
emotion_engine.py (Detect emotional tone)
    ↓
tts_service.py (Convert to speech with XTTS v2)
    ↓
voice_pool.py (Apply voice styling)
    ↓
audio_stitching.py (Combine audio segments)
    ↓
Output audio file → outputs/ directory


DEPENDENCIES
============

Key Python Packages:
- FastAPI/Flask: Web framework
- Pydantic: Data validation
- TTS (coqui-tts): Text-to-speech engine
- PyPDF2/pdfplumber: PDF processing
- Transformers: NLP models
- Torch: Deep learning framework
- Librosa: Audio processing
- Loguru: Logging
- Python 3.8+


ENVIRONMENT SETUP
=================

1. Create virtual environment: python3 -m venv venv
2. Activate: source venv/bin/activate
3. Install dependencies: pip install -r requirements.txt
4. Configure: cp .env.example .env (and edit .env)
5. Run: bash start.sh

NOTES
=====

- Models directory contains large pre-trained ML models (not uploaded to GitHub)
- Data directories are configured with .gitkeep to preserve structure
- Logs directory stores application runtime logs
- All Python cache files (__pycache__) are ignored
- Virtual environment (venv) is excluded from version control
